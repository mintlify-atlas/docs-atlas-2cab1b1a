Svix makes it easy to send webhooks from your application. You make one API call with a payload, and Svix handles delivery, retries, and security. This guide walks you through installing the SDK, creating an application, and sending your first message.

## Prerequisites

You need a Svix API token. You can get one from the [Svix dashboard](https://dashboard.svix.com) or, if you're self-hosting, generate one using `svix-server jwt generate`.

<Steps>

<Step title="Install the SDK">

Install the Svix client library for your language.

<CodeGroup>

```bash npm
npm install svix
```

```bash yarn
yarn add svix
```

```bash pip
pip install svix
```

```bash go get
go get github.com/svix/svix-webhooks/go
```

</CodeGroup>

</Step>

<Step title="Initialize the client">

Create a `Svix` instance with your API token. The client automatically routes to the correct regional endpoint based on the token suffix.

<CodeGroup>

```typescript TypeScript
import { Svix } from "svix";

const svix = new Svix("YOUR_AUTH_TOKEN");
```

```python Python
from svix import Svix

svix = Svix("YOUR_AUTH_TOKEN")
```

```go Go
import (
    "log"
    svix "github.com/svix/svix-webhooks/go"
)

sv, err := svix.New("YOUR_AUTH_TOKEN", nil)
if err != nil {
    log.Fatal(err)
}
```

</CodeGroup>

<Note>
If you are self-hosting Svix, pass a `serverUrl` option pointing to your instance:

<CodeGroup>

```typescript TypeScript
const svix = new Svix("YOUR_AUTH_TOKEN", {
  serverUrl: "http://localhost:8071",
});
```

```python Python
from svix import Svix, SvixOptions

svix = Svix("YOUR_AUTH_TOKEN", SvixOptions(server_url="http://localhost:8071"))
```

```go Go
import (
    "net/url"
    svix "github.com/svix/svix-webhooks/go"
)

serverUrl, _ := url.Parse("http://localhost:8071")
sv, err := svix.New("YOUR_AUTH_TOKEN", &svix.SvixOptions{
    ServerUrl: serverUrl,
})
```

</CodeGroup>
</Note>

</Step>

<Step title="Create an application">

An application represents a single user or tenant in your system. Each application has its own set of endpoints that receive webhooks.

<CodeGroup>

```typescript TypeScript
const app = await svix.application.create({ name: "My Application" });
console.log(app.id); // app_...
```

```python Python
from svix import ApplicationIn

app = svix.application.create(ApplicationIn(name="My Application"))
print(app.id)  # app_...
```

```go Go
import (
    "context"
    "fmt"
    svix "github.com/svix/svix-webhooks/go"
    "github.com/svix/svix-webhooks/go/models"
)

app, err := sv.Application.Create(context.Background(), models.ApplicationIn{
    Name: "My Application",
}, nil)
if err != nil {
    log.Fatal(err)
}
fmt.Println(app.Id) // app_...
```

</CodeGroup>

<Tip>
Supply an optional `uid` to tie the application to an ID from your own database. This lets you look it up later without storing Svix's generated ID.

<CodeGroup>

```typescript TypeScript
const app = await svix.application.create({
  name: "My Application",
  uid: "user_12345",
});
```

```python Python
app = svix.application.create(
    ApplicationIn(name="My Application", uid="user_12345")
)
```

```go Go
uid := "user_12345"
app, err := sv.Application.Create(context.Background(), models.ApplicationIn{
    Name: "My Application",
    Uid:  &uid,
}, nil)
```

</CodeGroup>
</Tip>

</Step>

<Step title="Send a message">

A message is a single webhook event dispatched to all endpoints registered under the application. The required fields are `eventType` (a dot-separated string naming the event) and `payload` (the JSON body to deliver).

<CodeGroup>

```typescript TypeScript
const msg = await svix.message.create(app.id, {
  eventType: "invoice.paid",
  payload: {
    id: "invoice_123",
    status: "paid",
    amount: 4900,
    currency: "usd",
  },
});
console.log(msg.id); // msg_...
```

```python Python
from svix import MessageIn

msg = svix.message.create(
    app.id,
    MessageIn(
        event_type="invoice.paid",
        payload={
            "id": "invoice_123",
            "status": "paid",
            "amount": 4900,
            "currency": "usd",
        },
    ),
)
print(msg.id)  # msg_...
```

```go Go
payload := map[string]any{
    "id":       "invoice_123",
    "status":   "paid",
    "amount":   4900,
    "currency": "usd",
}

msg, err := sv.Message.Create(context.Background(), app.Id, models.MessageIn{
    EventType: "invoice.paid",
    Payload:   payload,
}, nil)
if err != nil {
    log.Fatal(err)
}
fmt.Println(msg.Id) // msg_...
```

</CodeGroup>

Svix immediately dispatches the message to all of the application's registered endpoints and retries on failure using the configured retry schedule.

<Note>
You can optionally set `eventId` to a string of your choice for idempotent delivery. If a message with the same `eventId` already exists for the application, Svix returns a 409 conflict error rather than sending a duplicate.
</Note>

</Step>

</Steps>
