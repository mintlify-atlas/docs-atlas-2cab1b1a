---
title: "Endpoints"
description: "Manage the URLs where Svix delivers webhooks for each application."
---

An **endpoint** is a URL belonging to one of your customers where Svix sends webhook deliveries. Each endpoint is scoped to a single application. When you send a message to an application, Svix dispatches it to all active endpoints in that application that match the message's event type and channels.

## Creating an endpoint

The only required field when creating an endpoint is `url`. All other fields are optional.

<CodeGroup>

```typescript typescript
import { Svix } from "svix";

const svix = new Svix("YOUR_AUTH_TOKEN");

const endpoint = await svix.endpoint.create(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  {
    url: "https://api.example.com/webhooks",
    description: "Production webhook handler",
  }
);

console.log(endpoint.id); // ep_2MBCdZvPxmRfCa1GWXcXnS
```

```python python
from svix.api import SvixAsync
from svix.models import EndpointIn

svix = SvixAsync("YOUR_AUTH_TOKEN")

endpoint = await svix.endpoint.create(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    EndpointIn(
        url="https://api.example.com/webhooks",
        description="Production webhook handler",
    ),
)

print(endpoint.id)  # ep_2MBCdZvPxmRfCa1GWXcXnS
```

```go go
import (
    svix "github.com/svix/svix-webhooks/go"
    "github.com/svix/svix-webhooks/go/models"
)

svixClient, _ := svix.New("YOUR_AUTH_TOKEN", nil)

desc := "Production webhook handler"
endpoint, err := svixClient.Endpoint.Create(
    ctx,
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    models.EndpointIn{
        Url:         "https://api.example.com/webhooks",
        Description: &desc,
    },
    nil,
)
```

</CodeGroup>

### Filtering by event type

To receive only specific event types, set `filterTypes` to a list of event type names. An endpoint without `filterTypes` receives all event types.

<CodeGroup>

```typescript typescript
const endpoint = await svix.endpoint.create(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  {
    url: "https://api.example.com/webhooks/billing",
    description: "Billing event handler",
    filterTypes: ["invoice.created", "invoice.payment_failed", "subscription.cancelled"],
  }
);
```

```python python
endpoint = await svix.endpoint.create(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    EndpointIn(
        url="https://api.example.com/webhooks/billing",
        description="Billing event handler",
        filter_types=["invoice.created", "invoice.payment_failed", "subscription.cancelled"],
    ),
)
```

```go go
desc := "Billing event handler"
endpoint, err := svixClient.Endpoint.Create(
    ctx,
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    models.EndpointIn{
        Url:         "https://api.example.com/webhooks/billing",
        Description: &desc,
        FilterTypes: []string{"invoice.created", "invoice.payment_failed", "subscription.cancelled"},
    },
    nil,
)
```

</CodeGroup>

## Updating an endpoint

Use `update` to replace all fields, or `patch` to change only specific ones.

<CodeGroup>

```typescript typescript
// Full update
await svix.endpoint.update(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS",
  {
    url: "https://api.example.com/webhooks/v2",
    description: "Updated production handler",
    filterTypes: ["user.created", "user.deleted"],
  }
);

// Partial update — only update description
await svix.endpoint.patch(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS",
  { description: "Updated production handler" }
);
```

```python python
from svix.models import EndpointUpdate, EndpointPatch

# Full update
await svix.endpoint.update(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",

        url="https://api.example.com/webhooks/v2",
        description="Updated production handler",
        filter_types=["user.created", "user.deleted"],
```

## Disabling and enabling endpoints

Set `disabled: true` to stop delivery to an endpoint without deleting it. Set `disabled: false` to resume delivery.

<CodeGroup>

```typescript typescript
// Disable
await svix.endpoint.patch(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS",
  { disabled: true }
);

// Re-enable
await svix.endpoint.patch(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS",
  { disabled: false }
);
```

```python python
from svix.models import EndpointPatch

# Disable
await svix.endpoint.patch(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    EndpointPatch(disabled=True),
)

# Re-enable
await svix.endpoint.patch(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    EndpointPatch(disabled=False),
)
```

```go go
disabled := true
_, err = svixClient.Endpoint.Patch(
    ctx,
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",

```

## Signing secrets

Every endpoint has a signing secret used to generate the `svix-signature` header on outgoing webhook requests. Your customers use this secret to verify that webhooks originate from Svix.

### Getting the signing secret

<CodeGroup>

```typescript typescript
const secret = await svix.endpoint.getSecret(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS"
);

console.log(secret.key); // whsec_...
```

```python python
secret = await svix.endpoint.get_secret(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
)

print(secret.key)  # whsec_...
```

```go go
secret, err := svixClient.Endpoint.GetSecret(
    ctx,
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
)

fmt.Println(secret.Key)  // whsec_...
```

</CodeGroup>

### Rotating the signing secret

When you rotate a secret, the **previous secret remains valid for 24 hours**. This gives your customer time to update their verification code before the old secret expires.

<CodeGroup>

```typescript typescript
// Rotate to an auto-generated secret (recommended)
await svix.endpoint.rotateSecret(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS",
  {}
);

// Rotate to a specific secret you supply
await svix.endpoint.rotateSecret(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS",
  { key: "whsec_C2FVsBQIhrscChlQIMV+b5sSYspob7oD" }
);
```

```python python
from svix.models import EndpointSecretRotateIn

# Rotate to an auto-generated secret (recommended)
await svix.endpoint.rotate_secret(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    EndpointSecretRotateIn(),
)

# Rotate to a specific secret you supply
await svix.endpoint.rotate_secret(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    EndpointSecretRotateIn(key="whsec_C2FVsBQIhrscChlQIMV+b5sSYspob7oD"),
)
```

```go go
import "github.com/svix/svix-webhooks/go/models"

// Rotate to an auto-generated secret (recommended)
err = svixClient.Endpoint.RotateSecret(
    ctx,
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    models.EndpointSecretRotateIn{},
    nil,
)

// Rotate to a specific secret
key := "whsec_C2FVsBQIhrscChlQIMV+b5sSYspob7oD"
err = svixClient.Endpoint.RotateSecret(
    ctx,
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    models.EndpointSecretRotateIn{Key: &key},
    nil,
)
```

</CodeGroup>

<Note>
  Secrets must be base64-encoded random bytes and may optionally be prefixed
  with `whsec_`. When omitting the `key` field, Svix generates a secure secret
  automatically.
</Note>

## Custom headers

You can configure additional HTTP headers to be included in every webhook request sent to an endpoint. This is useful for passing authentication tokens or routing hints required by the receiving server.

### Setting headers

<CodeGroup>

```typescript typescript
await svix.endpoint.updateHeaders(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS",
  {
    headers: {
      Authorization: "Bearer my-secret-token",
      "X-Custom-Header": "my-value",
    },
  }
);
```

```python python
from svix.models import EndpointHeadersIn

await svix.endpoint.update_headers(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    EndpointHeadersIn(
        headers={
            "Authorization": "Bearer my-secret-token",
            "X-Custom-Header": "my-value",
        }
    ),
)
```

```go go
import "github.com/svix/svix-webhooks/go/models"

err = svixClient.Endpoint.UpdateHeaders(
    ctx,
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    models.EndpointHeadersIn{
        Headers: map[string]string{
            "Authorization":  "Bearer my-secret-token",
            "X-Custom-Header": "my-value",

    },
)
```

</CodeGroup>

## Recovering failed messages

When delivery fails after all retry attempts, you can trigger recovery to resend failed messages from a specific point in time. Only messages that ultimately failed (not those that eventually succeeded) are retried.

<CodeGroup>

```typescript typescript
import { Svix } from "svix";

const svix = new Svix("YOUR_AUTH_TOKEN");

const result = await svix.endpoint.recover(
  "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
  "ep_2MBCdZvPxmRfCa1GWXcXnS",
  { since: new Date("2024-01-01T00:00:00Z") }
);
```

```python python
from svix.models import RecoverIn
from datetime import datetime

result = await svix.endpoint.recover(
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    RecoverIn(since=datetime(2024, 1, 1)),
)
```

```go go
import (
    "time"
    "github.com/svix/svix-webhooks/go/models"
)

since := time.Date(2024, 1, 1, 0, 0, 0, 0, time.UTC)
result, err := svixClient.Endpoint.Recover(
    ctx,
    "app_2MBBmNNh1Ai7BJQ4aV96cL7bFJL",
    "ep_2MBCdZvPxmRfCa1GWXcXnS",
    models.RecoverIn{Since: since},
    nil,
)
```

</CodeGroup>

## Endpoint fields

<ResponseField name="url" type="string" required>
  The HTTPS URL where webhooks are delivered.
</ResponseField>

<ResponseField name="description" type="string">
  A human-readable label for the endpoint.
</ResponseField>

<ResponseField name="filterTypes" type="string[]">
  List of event type names this endpoint subscribes to. When omitted, the
  endpoint receives all event types.
</ResponseField>

<ResponseField name="channels" type="string[]">
  List of channels this endpoint subscribes to. When omitted, the endpoint
  receives messages from all channels.
</ResponseField>

<ResponseField name="disabled" type="boolean">
  When `true`, the endpoint is inactive and does not receive deliveries.
  Defaults to `false`.
</ResponseField>

<ResponseField name="uid" type="string">
  Optional user-supplied identifier for the endpoint. Can be used in place of
  the Svix-generated `id`.
</ResponseField>

<ResponseField name="metadata" type="object">
  A key-value map of arbitrary string metadata.
</ResponseField>
