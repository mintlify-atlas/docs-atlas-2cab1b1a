## Installation

```bash
pip install svix
```

The package is published as `svix` on PyPI (version `1.90.0`). It supports Python 3.8+ and provides both synchronous (`Svix`) and asynchronous (`SvixAsync`) clients.

## Initializing the client

<CodeGroup>

```python Sync
from svix import Svix

svix = Svix("AUTH_TOKEN")
```

```python Async
from svix import SvixAsync

svix = SvixAsync("AUTH_TOKEN")
```

</CodeGroup>

The client automatically detects the region suffix on your token (`.us`, `.eu`, `.in`, `.ca`, `.au`) and routes requests to the correct regional API endpoint.

### `SvixOptions`

```python
from svix import Svix, SvixOptions

options = SvixOptions(
    server_url="https://api.eu.svix.com",  # override region
    retry_schedule=[0.05, 0.1, 0.2],       # seconds between retries (max 5)
    timeout=15.0,                           # request timeout in seconds
)
svix = Svix("AUTH_TOKEN", options)
```

| Option           | Type            | Default             | Description                                              |
| :--------------- | :-------------- | :------------------ | :------------------------------------------------------- |
| `server_url`     | `str \| None`   | auto-detected       | Override the API base URL                                |
| `retry_schedule` | `list[float]`   | `[0.05, 0.1, 0.2]` | Seconds to wait before each retry attempt (max 5 items)  |
| `timeout`        | `float`         | `15.0`              | Request timeout in seconds                               |
| `proxy`          | `str \| None`   | `None`              | HTTP proxy URL                                           |
| `debug`          | `bool`          | `False`             | Enable debug logging                                     |

## Sending a webhook

<Steps>

### Create an application

```python
from svix.api import ApplicationIn

app = svix.application.create(ApplicationIn(name="my-app"))
print(app.id)  # app_...
```

### Send a message

```python
from svix.api import MessageIn

msg = svix.message.create(
    app.id,
    MessageIn(
        event_type="invoice.paid",
        payload={
            "type": "invoice.paid",
            "invoiceId": "invoice_123",
            "amount": 9900,
        },
    ),
)
print(msg.id)  # msg_...
```

</Steps>

### Async usage

The `SvixAsync` client exposes the same properties with `async/await` support:

```python
import asyncio
from svix import SvixAsync
from svix.api import ApplicationIn, MessageIn

async def main():
    svix = SvixAsync("AUTH_TOKEN")
    app = await svix.application.create(ApplicationIn(name="my-app"))
    msg = await svix.message.create(
        app.id,
        MessageIn(event_type="invoice.paid", payload={"type": "invoice.paid"}),
    )
    print(msg.id)

asyncio.run(main())
```

## Verifying a webhook

Use the `Webhook` class to validate signatures on incoming payloads.

```python
from svix import Webhook, WebhookVerificationError

wh = Webhook("whsec_<your-endpoint-secret>")

# In a Flask/FastAPI handler — pass the raw bytes and headers dict:
try:
    payload = wh.verify(request.body, {
        "svix-id": request.headers["svix-id"],
        "svix-timestamp": request.headers["svix-timestamp"],
        "svix-signature": request.headers["svix-signature"],
    })
    print("Verified payload:", payload)
except WebhookVerificationError:
    # Return 400 to the caller
    ...
```

<Warning>
  Pass the **raw, unmodified** request body bytes to `verify`. Parsing the body
  as JSON before verification will cause signature mismatches.
</Warning>

## Available client properties

| Property                           | Description                                        |
| :--------------------------------- | :------------------------------------------------- |
| `svix.application`                 | Create and manage applications                     |
| `svix.authentication`              | Generate dashboard access tokens                   |
| `svix.background_task`             | Track long-running background jobs                 |
| `svix.connector`                   | Manage connectors                                  |
| `svix.endpoint`                    | Create, update, and delete webhook endpoints       |
| `svix.environment`                 | Import and export environment configuration        |
| `svix.event_type`                  | Manage event type schemas                          |
| `svix.ingest`                      | Manage ingest sources                              |
| `svix.integration`                 | Manage third-party integrations                    |
| `svix.message`                     | Send webhook messages                              |
| `svix.message_attempt`             | Inspect and retry delivery attempts                |
| `svix.operational_webhook`         | Subscribe to Svix operational events               |
| `svix.operational_webhook_endpoint`| Manage operational webhook endpoints               |
| `svix.statistics`                  | Aggregate usage statistics                         |
| `svix.streaming`                   | Manage event streams                               |

All properties on `SvixAsync` have the same names and return coroutines.
